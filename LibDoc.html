Main:

This page is not intended to be an exhaustive libmesh-documentation nor a complete reference list. It moreover tries to give an introduction to the main structure of the library and give links to other pages for more detailed information.

In general, libmesh is a finite element (FEM) library If not mentioned differently, the functions and variables belong to the namespace "libmesh".

Libmesh, moreover, uses many external packages.

Functionality

One of the main figureheads of libmesh is their adaptive grid refinement, but there are some other functions that are quite useful for several applications.

For some purposes, libmesh offers several options as well. I will try to compare them and find differences between the packages.

Libmesh is build on top of several other libraries and for many applications further libraries need to be installed. One of these applications are eigen systems that require slepc which again needs petsc to be installed and configured accordingly. This means for example that petsc needs to be installed such that it uses complex numbers if this is needed for the libmesh-application.

Generation of Meshes

The tool tetgen can generate different types of meshes and Delaunay triangulations. That are, for example, Voronoi meshes.

Libmesh provides qhull as well. This is mentioned only at few places (the code is in 'contrib/qhull'. This library can be allowed when installing libmesh by the option '--enable-qhull'

Find out differences between qhull and tetgen!

infinite elements

The libraries for infinite elements are not compiled by default but require an extra option in the configure script.

Related to this option, there are the libraries inf_fe.h and libmesh/inf_elem_builder.h.

The libraries

The Basic libraries

The most important library is "libmesh/libmesh.h". Its most important functions are the initialisation (and closing) of the library and other libraries depending on it, such as mpi and PETSc.

The second basic library that is needed always is mesh.h that provides the class mesh

Implicit and Explicit System

Libmesh provides the libraries implicit_system.h , linear_implicit_system.h , nonlinear_implicit_system.h and explicit_system.h. These libraries contain each a class of same name.

The implicit systems, moreover, have an own solver class which contain different functions to solve liner/nonlinear systems and eigen equations using Laspack, PETSc or other libraries.

For nonlinear systems, various methods are provided by petsc in addition to solvers implemented in libmesh.

I didn't find any difference between explict and implicit systems yet.

Also, I don't really know what implicit_system.h does. It doesn't seem to be a relict; but I didn't find any real functionality for it.

An other important library for a class of systems is eigen_system.h which is needed e.g. to solve the schroedinger equation for bound particles.

Libraries for in- and output

There are several libraries that handle input and output of meshes and related quantities in different formats.

GetPot

The tool GetPot is a commandline- and inputfile parser supported by libmesh.

exodus

One of the formats of choise is Exodus 2 format (.exd) which is a binary format that can be converted to ascii using the tool 'ncdump'. This library is imported by libmesh/exodusII_io.h that provides an extra class exodusII_IO.

xdr

The second format that is important for in- and output is the file format External Data Representation 'xdr'. This is obtained by an option in installation (by default it seems to be on).

Note: xdr is supported by other programms and languages such as R as well.

This file format can be an elegant way to insert a mesh and maybe even an external potential to application programms.

tecplot

Such as exodus, tecplot can handle data in binary as well as ascii format.

miscelangelous

There also exists a library gnuplot_io.h which is able to handle 1d data only. However, it can be used to create nice gnuplot script and data files.

PETSc

The library PETSc is a very powerful tool for vector- and matrix operations. The developers of PETSc call it "a suite of data structures and routines for the scalable (parallel) solution of scientific applications modeled by partial differential equations." It provides many different solvers for linear systems where most of them work for complex matrices as well. However, many of these functions depend on external packages themselves. One important note about this library is that it is not only parallelized in terms of mpi, but supports cuda and opencl as well.

PETSc provides a detailed documentation and helps on its homepage.
